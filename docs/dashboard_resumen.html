Tecnicas y herramientas

Que aporta cada pieza al proyecto

Interfaz web con Gradio

Permite crear la aplicacion desde Python, organizar pestanas y recibir imagenes, capturas de webcam y videos desde el navegador.

UI app.py Spaces

Deteccion con MediaPipe Pose

Usa un modelo preentrenado para detectar landmarks corporales sin entrenar una red neuronal propia.

Pose 33 landmarks modelo preentrenado

Procesamiento con OpenCV

Lee frames de video, convierte entre BGR y RGB, escribe el video anotado y facilita el trabajo con archivos visuales.

Video BGR/RGB cv2

Representacion con NumPy

Las imagenes se manejan como arreglos numericos, lo que permite normalizar tipos, dimensiones y canales antes del analisis.

Arrays uint8 RGB

Metricas geometricas

Calcula inclinacion de hombros, inclinacion de cadera, alineacion troncal y angulos articulares usando puntos detectados.

atan2 umbrales educativo

Compatibilidad de video

Convierte la salida a MP4 H.264 con formato yuv420p para mejorar la reproduccion en Gradio y navegadores.

H.264 ffmpeg faststart

Flujo de procesamiento

Del archivo cargado al resultado mostrado
1

Entrada

El usuario carga imagen, captura webcam o sube un video corto.

2

Normalizacion

La imagen se valida como RGB y tipo uint8 para evitar errores.

3

Pose

MediaPipe detecta landmarks corporales y conexiones del esqueleto.

4

Metricas

Se calculan inclinaciones, alineacion y angulos articulares.

5

Salida visual

Se dibujan landmarks sobre imagen o frames del video.

6

Feedback

Gradio muestra tabla de metricas y devolucion educativa.

Reproducibilidad y despliegue

  • Hugging Face Spaces usa el frontmatter del README para ejecutar la app con Gradio.
  • packages.txt declara dependencias del sistema como ffmpeg y librerias graficas.
  • Dockerfile deja una alternativa opcional para ejecutar el proyecto en un contenedor.
  • Las versiones fijadas reducen errores por dependencias transitivas demasiado nuevas.

Criterios de lectura

Alcance y limites del analisis

Fortalezas tecnicas

  • Separacion clara entre interfaz, deteccion, metricas y video.
  • Uso de modelo preentrenado para concentrar el trabajo en integracion y analisis.
  • Conversion explicita entre RGB y BGR para evitar colores alterados.
  • Feedback docente orientado a interpretar resultados sin diagnosticar.

Limitaciones conocidas

  • No realiza diagnostico medico ni evaluacion clinica.
  • La deteccion depende de iluminacion, encuadre, visibilidad y una sola persona en escena.
  • Los umbrales son educativos y no reemplazan criterios profesionales.
  • Los videos largos o pesados pueden superar los recursos esperados del despliegue.
